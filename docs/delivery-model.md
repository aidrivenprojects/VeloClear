# VeloClear Delivery Model

## The Correct Flow

```text
Discovery
→ Initiation
→ Planning
→ Execution
→ Monitoring & Controlling
→ Closure
→ Learning
```

## Discovery

Discovery has two paths:

### Upload-based discovery
For users with:
- PDFs
- SOPs
- PM toolkits
- Excel trackers
- process maps
- forms
- BRDs

### Guided interview discovery
For users who only have the process in their head.

The guided interview captures:
- business problem
- users
- workflow
- approvals
- data fields
- exceptions
- reports
- risks
- success criteria

## Planning Order

For software delivery, the order is:

```text
Requirements
→ Product Vision
→ Epics
→ Features
→ Product Backlog
→ Prioritisation
→ Release Plan
→ Sprint Planning
→ Sprint Backlog
```

Product Backlog always comes before Sprint Backlog.

## Retrospectives

Retrospectives are not only closure artifacts.

They belong in Execution as a continuous sprint learning loop.

Closure contains final lessons learned and project-level learning consolidation.

## Intelligence Chain

```text
Blocked PBI
→ Sprint Slip
→ Dependency Risk Trigger
→ Issue Created
→ Escalation Raised
→ Decision Required
→ Resolution Logged
→ Retro Action Created
→ Future Recurrence Checked
```

This chain is the VeloClear moat.
